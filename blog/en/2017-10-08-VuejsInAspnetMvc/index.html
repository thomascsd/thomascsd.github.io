Things to note when using vue.js in asp.net MVC

I recently started using vue.js in my company's project, which is developed with asp.net mvc, so I found some small things to pay attention to when I couldn't use vue.js on .cshtml.

The first one is the shorthand for event binding such as @click, because it will conflict with the original Razor syntax, so you need to use the full writing method.

The following syntax will cause an error

<button @click=""></button>

You should use the complete syntax

<button v-on:click=""></button>

The second is the shorthand for data binding of attribure, for example, when you want to specify the path of the picture, you can write it like this

<img :src="defaultUrl" />

But when the format document is executed, the colon (:) will disappear, which will cause a strange bug. It took me an hour to find the cause, so it's OK to write it in the following syntax.

<img v-bind:src="defaultUrl" />
