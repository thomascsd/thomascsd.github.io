Experience of building a blog with Nuxt.js and Nuxtent

Nuxt.js is a vue.js framework with built-in SSR. The part I'm most interested in is that it can output static HTML, so I can easily deploy the program to GitHub Pages and use modules to extend its functions.

When I saw the Nuxtent module, which is a module that can get Markdown content, I wanted to rewrite the blog with Nuxt.js.

Nuxtent Template

For a quick start, you can use the Nuxtent Template to create a basic website structure. You need to install vue-cli in advance.

vue init nuxt-community/nuxtent-template my-site

Website Architecture

page

Create website routes. For example, if you create about.vue in the directory, the URL will be localhost:3000/about.

static

Static resources, such as images, the path will be based on the root directory, for example /images/bg.png

content

A specific directory for the Nuxtent module, where Markdown files are placed, that is, where articles are placed.

layout

The default layout is layout.vue. You can also customize the layout and then use the layout attribute to specify the custom layout.

export default {
  async asyncData({ app }) {
    return {
      posts: await app.$content('/').getAll(),
      name: 'Thomas Blog',
    };
  },
  layout: 'customLayout',
};

Configuration File

nuxtent.config.js

const Prism = require('prismjs');

module.exports = {
  content: {
    permalink: '/:slug',
    page: '/_content',
    generate: [
      // for static build
      'get',
      'getAll',
    ],
    isPost: true,
  },
  api: {
    baseURL: 'http://localhost:3200',
  },
  parsers: {
    md: {
      extend(config) {
        config.highlight = (code, lang) => {
          return `<pre class="language-${lang}"><code class="language-${lang}">${Prism.highlight(
            code,
            Prism.languages[lang] || Prism.languages.markup
          )}</code></pre>`;
        };
      },
    },
  },
};

content

  • permalink: Set the display format of the article path.

  • page: Specify the file name for displaying the article content. The default is _cotent, which points to _cotent.vue. According to the documentation, multiple files can be set.

  • isPost: I don't quite understand the usage of this attribute after reading the official documentation, but after reading the source code, I found that

    get date() {
      if (isDev || !cached.date) {
        const { filePath, fileName, section } = meta
        if (options.isPost) {
          const fileDate = fileName.match(/!?(\d{4}-\d{2}-\d{2})/) // YYYY-MM-DD
          if (!fileDate) {
            throw Error(`Post in "${section}" does not have a date!`)
          }
          cached.date = fileDate[0]
        } else {
          const stats = statSync(filePath)
          cached.date = dateFns.format(stats.ctime, 'YYYY-MM-DD')
        }
      }
      return cached.date
    }

When true, the markdown file name needs to be set to 2017-06-20-HelloWorld.md, and the date in the file name will be used as the article date.

When false, the file modification date will be used as the article date.

parser

I referred to the official documentation and used prismjs to add a colorizer function to the Mardown code.

Get Content

You can refer to the code of _content.vue and use get to get the content of the current path.

post: await app.$content('/').get(route.path);

If you want to get all articles, you can use the getAll method.

posts: await app.$content('/').getAll();

Comment Function

I want to use a comment function based on GitHub issues. I found that the gitment package just meets my needs. Installation and configuration are very simple.

<div class="row">
  <div class="col-lg-8 col-lg-offset-2 col-md-10 col-md-offset-1">
    <div id="comments"></div>
  </div>
</div>
import PostHeader from '../components/PostHeader';
import Gitment from 'gitment';

export default {
  async asyncData({ app, route, payload }) {
    return {
      post: await app.$content('/').get(route.path),
    };
  },
  mounted() {
    const gitment = new Gitment({
      id: this.post.title, // optional
      owner: 'thomascsd',
      repo: 'thomascsd.github.io',
      oauth: {
        client_id: 'client_id',
        client_secret: 'client_secret',
      },
    });

    gitment.render(document.getElementById('comments'));
  },
  components: {
    PostHeader,
  },
};

Deployment

When deploying static files to Github pages, I encountered some problems. According to the example settings, NODE_ENV='production' and baseUrl will become the path for formal deployment, for example, my blog URL thomascsd.github.io.

When using npm generate, the following error will appear

Finally, I found that the following steps are OK.

  • No need to add NODE_ENV
  • Set baseUrl to the default localhost:3200 and execute npm generate
  • Then use VSCode to search for localhost:3200, and you will find that the content of dist/api.js has baseURL:localhost:3200, and then replace the URL with the formal environment, such as my blog URL thomascsd.github.io
  • Finally, you can deploy to GitHub page
