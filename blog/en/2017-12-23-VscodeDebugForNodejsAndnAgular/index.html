Tips for Debugging Node.js and Angular with VSCode

After using Angular, I fell in love with TypeScript. I created a small project called angular-express-starter. Both the server and client are built with TypeScript. The client uses Angular CLI, and the server uses Express.js.

Because Node.js natively supports async/await in versions >8.0, and sequelize.js is also used in the project, a tsconfig.server.json is added, and the target is specified as ES6:

{
  "extends": "../tsconfig.json",
  "compilerOptions": {
    "outDir": "../out-tsc/server",
    "baseUrl": "",
    "target": "es6"
  }
}

The client is created by Angular CLI, and the target of tsconfig.json is specified as ES5:

{
  "compileOnSave": false,
  "compilerOptions": {
    "outDir": "./dist/out-tsc",
    "baseUrl": "src",
    "sourceMap": true,
    "declaration": false,
    "moduleResolution": "node",
    "emitDecoratorMetadata": true,
    "experimentalDecorators": true,
    "target": "es5",
    "typeRoots": ["node_modules/@types", "node_modules/typescript/lib"],
    "lib": ["es2016", "dom"]
  }
}

When debugging, you need to use ts-node to start Node.js, and ts-node has a parameter project that can specify the path of tsconfig.json. You can set it in the npm script like this:

"start": "ts-node --project server ./server/app.ts"

Node.js Debugging

In fact, VSCode has built-in Node.js debugging.

As shown in the figure above, select "Debug" > "Add Configuration", and then select "Node.js: Launch Program".

Because I use ts-node to start Node.js, I need to pass in additional parameters to specify the path of tscofig.json, but the following settings will cause errors because second-level parameters are not supported:

   {
      "name": "ts-node",
      "type": "node",
      "request": "launch",
      "args": ["${workspaceRoot}/server/index.ts"],
      "runtimeArgs": ["--nolazy", "-r", "ts-node/register", "-p server"],
      "sourceMaps": true,y
      "cwd": "${workspaceRoot}",
      "protocol": "inspector",
      "env": {
        "NODE_ENV": "dev"
      }

So I need to rewrite it in another way. First, create a js file named tshook.js with the following content:

require('ts-node').register({
  project: 'server/tsconfig.json',
});

Execute ts-node directly in tshook.js, and then specify the path of tsconfig.json. lauch.json is changed to the following:

{
  "name": "ts-node",
  "type": "node",
  "request": "launch",
  "args": ["${workspaceRoot}/server/index.ts"],
  "runtimeArgs": ["--nolazy", "-r", "${workspaceRoot}/tshook.js"],
  "sourceMaps": true,
  "cwd": "${workspaceRoot}",
  "protocol": "inspector",
  "env": {
    "NODE_ENV": "dev"
  }
}

Press F5 or the green play button to enter debug mode, which is very similar to debugging .Net programs in Visual Studio.

Angular Debugging

Refer to the official document Chrome Debugging with Angular CLI, there are a few things to note:

  1. Angular CLI needs to be 1.3 or above.

  2. The Debugger for Chrome extension needs to be 3.1.4 or above.

It should be noted that you need to use npm start to start the Angular application before debugging, and then you can press F5 or the green play button to enter the debugging mode.

Conclusion

  1. Debugging Node.js with VSCode is very similar to debugging .Net programs in Visual Studio. Press F5 to start the application.

  2. To debug an Angular application, you need to run ng serve to start Angular, and then debug the program with VSCode.
